Mobile Booking UX Audit
Why 89.7% of Mobile Users Never Select a Room
Based on frame-by-frame analysis of the actual Shelborne mobile booking flow.
5+ screen scrolls before first room. Header, step indicator, date bar, guest selectors, promo code, special rates, "add another room", and 4 filter dropdowns — all above the first room card.
Each room card = 3–4 full screens tall. Full-width photo + long description + "VIEW OFFERS" expand + 7 amenity icons + expanded rate details. Users must scroll ~30 screens to see all rooms.
10+ rooms listed equally — no hierarchy. No "Most Popular", no recommended default. Every room competes equally, creating decision paralysis on a tiny screen.
No CTA visible without scrolling inside each card. The "SELECT" button is buried below amenity icons and expanded offers — multiple swipes below the room image.
Taxes still hidden. Price shows "INCLUDES $75 RESORT FEE" but final taxes only appear after expanding offers.
